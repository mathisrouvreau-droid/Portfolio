Optimisation de la gestion du parc informatique

01 Présentation du projet

Conception et déploiement d'un réseau complet pour l'établissement MACODEMY, composé de 3 services segmentés en VLANs (Administratif, Enseignants, Etudiants), avec un serveur centralisé sous Windows Server 2019 et un serveur Web sous Debian en DMZ.

Services déployés sur le serveur Windows Server 2019

Active Directory

Gestion centralisée des utilisateurs, groupes et unités d'organisation

DHCP

Distribution automatique des adresses IP par VLAN (3 étendues)

DNS

Redirection macodemy.local vers le serveur Web en DMZ

GPO

Fond d'écran personnalisé + déploiement Firefox/Chrome automatique

Accès utilisateurs

  • Nom d'utilisateur : nomp (ex: rouvreaum)
  • Mot de passe initial : premco*963
  • Lecteur Perso : espace personnel par utilisateur
  • Répertoire commun : partagé au sein du service (lecture/écriture)
  • Ressources : admins = contrôle total, enseignants = lecture/écriture, étudiants = lecture seule

05 Serveur Web Debian

Serveur Apache2 en DMZ, IP statique 192.168.10.1, hébergeant le site interne de l'établissement.

Installation

Bash
sudo apt update sudo apt install apache2

Configuration du VirtualHost

/etc/apache2/sites-available/000-default.conf
ServerName macodemy.local DocumentRoot /var/www/html

Le site est accessible via http://macodemy.local ou http://www.macodemy.local depuis tous les VLANs grâce au DNS interne.

06 Configuration routeurs & switchs

Routeur Local — Sous-interfaces VLAN

Cisco IOS
interface GigabitEthernet0/0 ip address 172.16.1.254 255.255.255.0 interface GigabitEthernet0/0.10 encapsulation dot1Q 10 ip address 172.16.10.254 255.255.255.0 ip helper-address 172.16.1.1 interface GigabitEthernet0/0.20 encapsulation dot1Q 20 ip address 172.16.20.254 255.255.255.0 ip helper-address 172.16.1.1 interface GigabitEthernet0/0.30 encapsulation dot1Q 30 ip address 172.16.30.254 255.255.255.0 ip helper-address 172.16.1.1 ip route 0.0.0.0 0.0.0.0 172.16.255.254 ip route 192.168.10.0 255.255.255.0 172.16.255.254

ACL — Cloisonnement inter-VLAN

Cisco IOS
access-list 101 deny ip 172.16.30.0 0.0.0.255 172.16.10.0 0.0.0.255 access-list 101 permit ip any any access-list 103 permit tcp 172.16.0.0 0.0.255.255 host 192.168.10.1 eq www access-list 103 permit tcp 172.16.0.0 0.0.255.255 host 192.168.10.1 eq 443

Switch Local — VLANs

Cisco IOS
interface FastEthernet0/1 switchport access vlan 10 switchport mode access interface FastEthernet0/2 switchport access vlan 20 switchport mode access interface FastEthernet0/3 switchport access vlan 30 switchport mode access

Routeur DMZ

Cisco IOS
interface GigabitEthernet0/0 ip address 192.168.10.254 255.255.255.0 interface Serial0/0/0 ip address 172.16.255.254 255.255.255.252 router ospf 1 network 172.0.0.0 0.255.255.255 area 0 ip route 0.0.0.0 0.0.0.0 172.16.255.253

07 Intégration postes clients

Installation Windows 10

  1. Création d'une clé USB bootable via Rufus (ISO Windows 10 22H2)
  2. Démarrage sur clé USB via BIOS — installation en UEFI

Jonction au domaine

  1. Définir le DNS préféré : 172.16.1.1
  2. Joindre le domaine MACODEMY via Propriétés système
  3. Valider avec les identifiants administrateur du domaine
