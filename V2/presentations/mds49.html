01 Contexte & Objectifs

Conception d'une infrastructure réseau sécurisée et segmentée pour la Maison Départementale des Sports 49. Le réseau interne est divisé en 4 VLANs (Volley, Escrime, Administratif, Partenaires), avec un contrôleur de domaine AD, un serveur Web en DMZ accessible via NAT/PAT, une supervision Zabbix et un accès WiFi sécurisé par RADIUS.

Objectifs du projet

Segmentation

Isolation par VLAN : Volley, Escrime, Administratif, Partenaires

Services centralisés

AD / DHCP / DNS sur Windows Server 2019

Routage

Inter-VLAN + OSPF entre routeurs

DMZ

Serveur Web interne + accès externe via NAT/PAT

Supervision

Zabbix : agents + SNMP sur tous les équipements

WiFi sécurisé

WPA2-Enterprise via FreeRadius + AD

03 Routeurs (RDC & MDS49)

Routeur RDC — Sous-interfaces VLAN

Routage interne entre les 4 VLANs, DHCP Relay via ip helper-address, ACL de filtrage inter-VLAN, routage OSPF.

Cisco IOS — Sous-interfaces 802.1Q
interface GigabitEthernet0/0/0.10 encapsulation dot1Q 10 ip address 172.16.63.254 255.255.192.0 ip helper-address [IP_SERVEUR_DHCP] interface GigabitEthernet0/0/0.20 encapsulation dot1Q 20 ip address 172.16.127.254 255.255.192.0 ip helper-address [IP_SERVEUR_DHCP] interface GigabitEthernet0/0/0.30 encapsulation dot1Q 30 ip address 172.16.191.254 255.255.192.0 ip helper-address [IP_SERVEUR_DHCP] interface GigabitEthernet0/0/0.100 encapsulation dot1Q 100 ip address 172.16.255.254 255.255.192.0 ip helper-address [IP_SERVEUR_DHCP]

ACL inter-VLAN (Routeur RDC)

Cisco IOS
access-list 1 deny 172.16.0.0 0.0.64.255 access-list 1 permit any interface g0/0/1 ip access-group 1 in

Routeur MDS49 — Routeur central

Routage entre DMZ, entreprise et Internet. NAT/PAT + ACL standards et étendues. Routage OSPF.

InterfaceAdresseFonction
g0/0/1192.168.0.1/30Vers Routeur RDC
g0/0/010.54.0.254/24DMZ
s0/1/0100.100.100.1/30Vers Internet (FAI)

NAT/PAT

Translation de l'adresse publique 8.8.8.1:80 vers l'adresse privée 10.54.0.100:80 — permet l'accès externe au serveur Web en DMZ.

ACL configurées sur MDS49

  • 2 ACL standard : autorisent les sous-réseaux internes et le serveur Web à accéder à Internet (liées au NAT)
  • 1 ACL étendue (WEB) : permet à tous les hôtes d'accéder uniquement au service HTTP du serveur interne 10.54.0.100

05 Serveur AD / DHCP / DNS / GPO

Active Directory — MDS49.local

  • 4 OU : Bureaux_Administratifs, Comité_Volley, Comité_Escrime, Partenaires
  • Groupes de sécurité : un par OU (GR_Bureaux_Administratifs, GR_Comité_Volley...)
  • Nomenclature : nom + initiale prénom (ex: challainm)
  • Répertoires : Perso par utilisateur + Commun par service (droits différenciés)

DHCP

Distribution automatique des IP aux 4 VLANs via DHCP Relay (ip helper-address sur le routeur RDC). Deux réservations : serveur Zabbix et serveur Windows.

DNS

Résolution interne au domaine MDS49.local, installé automatiquement avec le rôle AD DS.

GPO

Interdiction du Panneau de configuration pour tous les utilisateurs :

Configuration utilisateur > Stratégies > Modèles d'administration > Panneau de configuration → "Interdire l'accès au Panneau de configuration et l'application Paramètres du PC"
