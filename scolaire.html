Projet MACODEMY — SIO 1ère année

Optimisation de la gestion du parc informatique

Contexte

Conception et déploiement d'un réseau complet pour l'établissement MACODEMY en équipe (Hugo, Mathis, Evan). Le réseau comprend 3 services (Administratif, Enseignants, Étudiants) segmentés en VLANs séparés, un serveur Windows Server 2019 centralisé et un serveur Web Debian en DMZ.

Équipe

Hugo PASDOIT — Mathis ROUVREAU — Evan RAIMBAULT

Environnement technique

Systèmes : Windows Server 2019 Active Directory DHCP / DNS GPO
Réseau : Cisco (routeurs & switchs) VLAN 802.1Q OSPF ACL
Serveurs : Debian Apache2 Windows 10 Packet Tracer

Activités réalisées — MACODEMY

🖥️

Serveur Windows Server 2019 — AD / DHCP / DNS

Déploiement d'un contrôleur de domaine macodemy.local avec gestion centralisée des utilisateurs par Unités d'Organisation, distribution automatique d'adresses IP par VLAN et résolution DNS interne.

Réalisations :
  • Création des OU : Administratif, Enseignants, Étudiants
  • Groupes de sécurité associés à chaque OU
  • Comptes utilisateurs (format nomp)
  • 3 étendues DHCP (VLAN 10, 20, 30) avec DHCP Relay
  • Zone DNS macodemy.local → serveur Web Debian (192.168.10.1)
Compétences SISR :
B1.2 B1.3 B1.5
📂

Répertoires réseau & Lecteurs partagés

Mise en place de répertoires personnels et communs pour chaque utilisateur, avec gestion fine des droits d'accès selon le groupe d'appartenance. Scripts d'ouverture de session pour le mapping automatique.

Réalisations :
  • Lecteur Perso (\\WINSERVACODEMY\Perso\%username%)
  • Lecteur Commun par service (Commun_Administratif, Commun_Enseignants, Commun_Etudiants)
  • Lecteur Ressources : contrôle total (admin), lecture/écriture (enseignants), lecture (étudiants)
  • Scripts de session dans NETLOGON (NET USE K: \\...)
Compétences SISR :
B1.2 B2.2
⚙️

GPO — Fond d'écran & déploiement logiciels

Configuration de stratégies de groupe pour personnaliser l'environnement utilisateur et automatiser le déploiement applicatif sur l'ensemble du parc.

Réalisations :
  • GPO fond d'écran personnalisé MACODEMY (via chemin réseau partagé)
  • GPO installation automatique de Mozilla Firefox
  • GPO installation automatique de Google Chrome
  • Application par Unité d'Organisation ciblée
Compétences SISR :
B1.2 B1.5
🌐

Infrastructure réseau — VLANs, OSPF & ACL

Segmentation complète du réseau par VLAN via switchs Cisco, routage dynamique OSPF entre les sites et ACL pour contrôler les flux inter-VLANs.

Réalisations :
  • VLAN 10 (Administratif), VLAN 20 (Enseignants), VLAN 30 (Étudiants)
  • Sous-interfaces 802.1Q sur routeur local (ip helper-address DHCP)
  • Routage OSPF entre routeur local et routeur DMZ
  • ACL étendues : isolement étudiants / accès web seul autorisé
  • Schéma réseau simulé sur Packet Tracer
Compétences SISR :
B3.2 B3.3 B4.1

Serveur Web Debian — Apache2 en DMZ

Installation et configuration d'un serveur web Apache2 sous Debian, placé en zone DMZ, accessible depuis tous les postes du réseau via le nom de domaine interne macodemy.local.

Réalisations :
  • IP statique : 192.168.10.1 (DMZ)
  • Installation Apache2 (apt install apache2)
  • Configuration VirtualHost (ServerName macodemy.local)
  • Déploiement des fichiers dans /var/www/html
  • Accès via http://macodemy.local depuis tous les VLANs
Compétences SISR :
B1.2 B3.2

Intégration des postes clients Windows 10

Installation et intégration au domaine de postes clients Windows 10 (22H2), un par VLAN, avec configuration DNS et jonction au domaine MACODEMY.

Réalisations :
  • Création de clés USB bootables via Rufus (ISO Windows 10)
  • Installation UEFI de Windows 10 (22H2)
  • Définition du DNS préféré : 172.16.1.1 (WinServ)
  • Jonction au domaine MACODEMY via les identifiants administrateur
  • Vérification des lecteurs réseau (K:, P:, R:) et du fond d'écran GPO

B1.1 B1.4

Projet PPE3 — MDS49 — SIO 2ème année

Infrastructure réseau sécurisée — Maison Départementale des Sports 49

Contexte

Conception et déploiement d'une infrastructure réseau sécurisée et segmentée pour la Maison Départementale des Sports 49 (MDS49). Le réseau est divisé en 4 VLANs (Volley, Escrime, Administratif, Partenaires), avec contrôleur de domaine AD, serveur web en DMZ, supervision Zabbix et authentification WiFi via RADIUS. Gestion de projet en méthode Agile (Trello).

Équipe

Mathis ROUVREAU — Evan RAIMBAULT — Julien TOUZEAU

Environnement technique

Systèmes : Windows Server 2019 Active Directory DHCP / DNS Debian
Réseau : Cisco (routeurs & switch) VLAN 802.1Q OSPF NAT/PAT ACL
Services : Zabbix SNMP FreeRadius WPA2-Enterprise Apache2

Activités réalisées — PPE3 MDS49

🔌

Infrastructure réseau multi-VLANs & routage OSPF

Segmentation du réseau MDS49 en 4 VLANs distincts. Le routeur RDC gère les sous-interfaces 802.1Q, le DHCP Relay et les ACL de filtrage. Les routes entre routeurs sont configurées en OSPF.

Réalisations :
  • VLAN 10 (Volley) — 172.16.0.0/18
  • VLAN 20 (Escrime) — 172.16.64.0/18
  • VLAN 30 (Administratif) — 172.16.128.0/18
  • VLAN 100 (Partenaires) — 172.16.192.0/18
  • Sous-interfaces 802.1Q sur Routeur RDC (g0/0/0.10, .20, .30, .100)
  • DHCP Relay (ip helper-address) par VLAN
  • ACL inter-VLAN + OSPF inter-routeurs
Compétences SISR :
B3.2 B3.3
🔄

Routeur central MDS49 — NAT/PAT & ACL étendues

Configuration du routeur de bordure assurant le routage entre la DMZ, le réseau interne et Internet, avec translation d'adresses NAT/PAT et filtrage par ACL étendues.

Réalisations :
  • NAT/PAT : adresse publique 8.8.8.1:80 → 10.54.0.100:80
  • ACL étendue WEB : autorisation HTTP seul vers serveur interne
  • 2 ACL standard liées au NAT (accès sous-réseaux + serveur web)
  • Interface DMZ (10.54.0.254/24), Réseau interne (192.168.0.1/30), Internet (s0/1/0)
Compétences SISR :
B3.3 B4.1
🖥️

Serveur Windows 2019 — AD / DHCP / DNS & GPO

Déploiement du contrôleur de domaine MDS49.local avec arborescence AD par comité, service DHCP multi-étendues, DNS interne et GPO de sécurité.

Réalisations :
  • OU : Bureaux_Administratifs, Comité_Volley, Comité_Escrime, Partenaires
  • Groupes de sécurité par OU avec répertoires personnels et communs
  • Droits différenciés : responsables (écriture) / employés (lecture) sur commun
  • DHCP : étendues par VLAN + réservations (Zabbix, Windows Server)
  • GPO : interdiction du panneau de configuration aux utilisateurs
Compétences SISR :
B1.2 B1.5 B2.2
🐧

Serveur Web Debian — Apache2 en DMZ

Hébergement du site institutionnel MDS49 sur un serveur Debian placé en DMZ (10.54.0.100), accessible depuis l'intérieur et l'extérieur via NAT/PAT.

Réalisations :
  • IP statique : 10.54.0.100 (DMZ 10.54.0.0/24)
  • Installation Apache2 (apt install apache2 -y)
  • Accès interne direct et accès externe via NAT/PAT (8.8.8.1:80)
  • ACL de filtrage en entrée de la DMZ
Compétences SISR :
B1.2 B3.2

Supervision Zabbix — Monitoring de l'infrastructure

Déploiement d'un serveur Zabbix pour surveiller en temps réel l'ensemble de l'infrastructure : serveurs, routeurs et switch via agents et protocole SNMP.

Réalisations :
  • Agent Zabbix installé sur serveur AD et serveur Web Debian
  • Supervision SNMP des routeurs Cisco (snmp-server community RO)
  • Supervision SNMP du switch (snmp-server community SW)
  • Tableau de bord : état serveurs, équipements réseau, alertes

  • Intégration AD via Winbind
  • Authentification 802.1X — contrôle d'accès dynamique via AD
